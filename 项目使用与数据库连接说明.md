# 股票数据抓取与管理项目使用说明

本项目主要用于自动从指定的 API 接口抓取股票相关的历史数据、实时数据、财务数据等，并统一存入本地的 ClickHouse 数据库中。
项目自带一个 Web 管理后台，方便直观地管理各种抓取任务、配置 API Key 并直接查询数据库中的数据。

## 一、项目启动

本项目基于 Docker 和 Docker Compose 进行容器化部署。

1. **环境准备**
   确保你的服务器或本地机器已经安装了 `Docker` 和 `Docker Compose`。

2. **启动服务**
   在项目根目录（`docker-compose.yml` 所在目录）下执行以下命令启动所有服务：
   ```bash
   docker-compose up -d
   ```

   这会启动两个容器：
   - `stock_fetcher_ch`：ClickHouse 数据库。
   - `stock_fetcher_app`：Python 抓取脚本及 Web 管理后台。

3. **访问管理后台**
   服务启动后，在浏览器中访问：`http://localhost:8082`
   - 在后台可以进行 API Key 的配置。
   - 可以在后台点击启动或停止各类数据抓取任务（如历史数据、实时分时、WebSocket 推送等）。
   - 提供了一个可视化的 SQL 查询界面，方便随时查询数据。

## 二、配置与 API Key

- **获取权限**：使用本项目的抓取功能需要合法的 API Key。
- **配置方式**：推荐直接在 `http://localhost:8082` 的 Web 界面进行配置，配置成功后会自动保存到项目目录下的 `.env` 文件中。
- 所有的 Python 脚本在运行前都会检查 `.env` 文件中是否有有效的 API Key，如果没有或者提示 `401/403` 无权限错误，脚本将自动停止运行。

## 三、脚本功能说明

- `fetch_history.py`：用于获取指定的历史K线数据（分钟、日线）和财务数据，内置补缺逻辑和多进程加速。
- `fetch_realtime.py`：获取当天的实时快照数据。
- `fetch_realtime_minute.py`：定时获取实时分时数据，并自动补缺。
- `fetch_ws.py`：通过 WebSocket 长连接接收实时的股票行情推送数据。
- `web_server.py`：FastAPI 编写的 Web 服务端，用于承载管理界面和任务调度。

# ClickHouse 数据库连接说明

项目部署后，ClickHouse 数据库通过 Docker 容器运行，并映射了端口供外部工具（如 DBeaver, DataGrip, Navicat 等）或程序进行连接。

## 1. 默认连接信息

- **主机地址 (Host)**: 部署该 Docker 容器的服务器 IP。如果在本地运行，则是 `127.0.0.1` 或 `localhost`。
- **HTTP 端口 (HTTP Port)**: `8126` （主要用于部分 BI 工具或直接发 HTTP 请求查询）。
- **TCP/Native 端口 (TCP Port)**: `9003` （主要用于 DBeaver/DataGrip 或 Python/Java 等原生驱动连接）。
- **数据库名 (Database)**: `stock_data`
- **用户名 (User)**: `default`
- **密码 (Password)**: *（默认为空，无密码）*

## 2. 内部网络连接 (Docker 内部)
如果你的其他脚本同样在这个 Docker 的默认网络里，或者是本项目内部的 Python 脚本，应使用内部通信参数连接：
- **Host**: `clickhouse`
- **Port**: `9000`

## 3. 使用第三方工具连接示例 (以 DBeaver 为例)

1. 打开 DBeaver，点击新建连接，选择 `ClickHouse`。
2. 在 **Host** 栏输入你的服务器 IP（如 `127.0.0.1`）。
3. 在 **Port** 栏输入 `8126` （如果使用 HTTP 驱动）或 `9003`（如果使用 Native 驱动，具体取决于你的 DBeaver 驱动设置）。
4. 在 **Database/Schema** 栏输入 `stock_data`。
5. 在 **Username** 栏输入 `default`。
6. **Password** 栏留空。
7. 点击 "Test Connection" 测试连接是否成功即可。
